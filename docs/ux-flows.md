## 2. 日常记账流程

```
日常场景: 用户消费了一笔钱, 打开应用记录
────────────────────────────────────────

1. 打开应用 (已有 Token, 自动进入仪表盘)
2. 确认当前账本 (头部 Select 切换)
3. 点击侧边栏"交易记录"或点击"+ 新增"按钮
4. 填写表单:
   a. 选择类型 (支出/收入)
   b. 选择分类 (自动过滤对应类型)
   c. 输入金额
   d. 选择币种 (默认 CNY, 切换币种时自动折算)
   e. 输入描述 (可选)
   f. 确认日期 (默认今天)
   g. 输入标签 (可选)
5. 点击确定 → 提交 → 成功提示
6. 回到仪表盘查看更新后的统计

平均耗时: 15-30 秒 (熟练后)
```

## 3. 创建账本流程

```
场景: 用户新增一本分类账本 (例如旅行账本)
────────────────────────────────────────

1. 登录后进入侧边栏"账本管理"
2. 账本管理页面 (如无账本, 显示空状态 + 创建按钮)
3. 点击"新建账本"
4. 填写:
   a. 账本名称 (必填, 如 "旅行账本")
   b. 描述 (可选, 如 "日本旅行期间开销")
   c. 本位币 (默认 CNY, 可选 USD/EUR 等)
   d. 图标 (可选 Emoji, 如 "✈️")
   e. 颜色 (可选, 用于卡片高亮)
5. 点击确定 → POST /ledgers → 成功
6. 新账本出现在列表中
7. 头部账本选择器可选到新账本
8. 切换到该账本 → 创建分类/记录交易
```

## 4. 分类管理流程

```
场景: 用户自定义分类体系
────────────────────────

### 4.1 创建分类

1. 侧边栏 → "分类管理"
2. 切换到"支出分类"或"收入分类" Tab
3. 点击"新建分类"
4. 填写:
   a. 名称 (如 "外卖")
   b. 图标 (Emoji, 如 "📦")
   c. 颜色 (可选)
   d. 排序 (决定显示顺序)
5. 点击确定 → 创建成功
6. 分类出现在列表中

### 4.2 创建子分类

1. 在分类列表点击编辑已有父分类
2. 创建分类时通过 API 设置 `parent_id`
   (当前前端 UI 暂未暴露子分类创建入口, 需后端直接调用)
```

## 5. 多币种记账流程

```
场景: 用户持有美元收入, 记账到人民币本位币账本
────────────────────────────────────────

1. 前置条件: 已录入 USD→CNY 汇率
   - 进入"汇率管理" → 新增汇率
   - from: USD, to: CNY, rate: 7.24, date: 2026-05-26

2. 进入"交易记录" → 新增交易

3. 填写:
   - 类型: 收入
   - 分类: 工资
   - 金额: 5000
   - 币种: USD ▼ (切换到美元)
   - 描述: Freelance project payment

4. 提交时后端自动:
   - 查询 USD→CNY 最新汇率 (DB → Cache)
   - 计算 base_amount = 5000 × 7.24 = 36,200.00
   - 存储: amount=5000, currency=USD, base_amount=36200

5. 列表中显示: +$5000.00 (CNY ¥36,200.00)
```

## 6. 查看报表流程

```
### 6.1 仪表盘 (当前)

1. 侧边栏 → "仪表盘"
2. 页面显示:
   - 三个统计卡片: 总收入 / 总支出 / 结余
   - 最近 5 条交易记录
   - 支出分类排行 (按金额降序, 每项含金额+笔数)
3. 点击统计卡片跳转到对应的交易列表 (带筛选)
4. 点击"查看全部"跳转到交易列表

### 6.2 仪表盘 (v2.0)

1. 月度趋势折线图
   - 位于统计卡片下方
   - 显示近 6 个月收入/支出趋势
   - 红色线 = 支出, 绿色线 = 收入
   - 鼠标悬停显示具体数值 (Tooltip)
   - 数据来自 `GET /monthly-trend?months=6`

2. 分类支出环形图
   - 位于折线图下方
   - 默认显示支出分布, 可通过下拉切换为收入
   - 前 8 大分类 + "其他" 合并项
   - 每项显示: 图标 + 分类名 + 金额 + 百分比
   - 点击扇形跳转到对应分类的交易列表 (带分类筛选)
   - 支持日期范围筛选 (本年/本月/自定义)
   - 数据来自 `GET /category-breakdown?type=expense`
```

## 7. 数据导入导出流程

```
### 7.1 导出

1. 进入"交易记录"页面
2. 设置筛选条件 (可选: 类型、日期范围、分类)
3. 点击"导出" → 下拉选择 CSV 或 JSON
4. 应用当前筛选, 下载文件
   - 文件名: `{账本名}_交易记录_{日期范围}.csv` 或 `.json`
5. 如数据量 > 5000, 后台异步处理, 完成后推送下载链接

### 7.2 导入

1. 进入"交易记录"页面
2. 点击"导入" → 打开导入 Modal
3. 步骤:
   a. 下载导入模板 (可选) → 获取模板 CSV/JSON 格式示例
   b. 选择 CSV 或 JSON 文件上传
   c. 系统解析并预览前 5 条数据
   d. 用户确认数据正确后提交
   e. 系统执行批量导入 (异步)
   f. 完成后提示导入结果 (成功条数 / 失败条数 + 错误详情)
```

## 8. 设置页操作流程

```
### 8.1 用户信息查看

1. 侧边栏 → "设置"
2. 显示:
   - 用户名 (禁用, 只读)
   - 邮箱 (只读显示)
   - 用户 ID (只读显示)
   - 注册时间 (只读显示)
3. 信息来自 `GET /auth/me`

### 8.2 修改密码

1. 在设置页点击"修改密码"按钮 → 弹出 Modal
2. 表单:
   a. 旧密码 (必填, password 类型)
   b. 新密码 (必填, 6-100 字符)
   c. 确认新密码 (必填, 必须与新密码一致)
3. 点击确定 → `PUT /auth/password`
4. 成功 → 提示"密码已修改, 请重新登录"
5. 失败 → 显示错误信息 (旧密码错误/新密码过短等)
6. 成功后自动跳转到登录页

### 8.3 修改邮箱

1. 在设置页点击"修改邮箱"按钮 → 弹出 Modal
2. 表单:
   a. 新邮箱 (必填, 有效邮箱格式)
3. 点击确定 → `PUT /auth/email`
4. 成功 → 提示"邮箱已修改"并即时生效
5. 失败 → 显示错误信息 (邮箱格式错误/已存在)
```

## 11. 日历视图操作流程

```
场景: 用户查看某个月的每日收支情况
─────────────────────────────────────

1. 侧边栏 → "日历视图"
2. 默认显示当前月份的日历 (CalendarViewPage.tsx)
3. 日历每个日期格子显示:
   - 支出金额 (红色, 如 "-¥1,200")
   - 收入金额 (绿色, 如 "+¥5,000")
   - 交易笔数 (灰色, 如 "3笔")
   - 无交易则显示空白
4. 操作:
   a. 点击"<" 上一月 / ">" 下一月 按钮切换月份
   b. 或点击月份标题下拉选择具体月份
   c. 切换时调用 `GET /daily-transactions?year=2026&month=5`
5. 点击某个日期:
   - 弹出侧边 Drawer 或 Modal
   - 显示该日全部交易明细 (按时间降序)
   - 每行: 分类图标 + 分类名 + 金额 + 描述 + 标签
   - 点击某条交易可跳转到交易记录列表
6. 顶部有"今天"按钮, 点击快速回到当前月
```

## 12. 批量操作流程

```
### 12.1 批量删除

场景: 用户发现一批错误记录, 需要批量删除
─────────────────────────────────────

1. 进入"交易记录"页面
2. 表格左侧出现复选框列 (未勾选时为空白方框)
3. 操作:
   a. 勾选表头全选复选框 → 选中当前页全部交易
   b. 或逐条勾选需要删除的交易
4. 选中 1 条以上后, 出现批量操作工具栏:
   ┌────────────────────────────────────────────────┐
   │ 已选 N 条  [批量删除]  [批量修改分类 ▼]        │
   └────────────────────────────────────────────────┘
5. 点击"批量删除":
   a. 弹出确认对话框:
      - 标题: "确认删除"
      - 内容: "确定要删除选中的 N 条交易记录吗? 此操作不可恢复。"
      - 按钮: [取消] [确认删除]
   b. 点击确认 → `POST /transactions/batch-delete` → `{ids: [...]}`
   c. 加载中状态 (按钮禁用 + Loading 图标)
6. 完成后:
   - 成功 → 提示 "已删除 N 条记录"
   - 部分失败 → 提示 "已删除 X 条, Y 条失败"
   - 表格刷新, 取消选中状态

### 12.2 批量修改分类

场景: 用户需要将一批交易重新归类
─────────────────────────────────────

1. 勾选需要修改的交易 (同 12.1 步骤 1-3)
2. 点击"批量修改分类"
3. 弹出 Modal:
   - 选择目标分类 (下拉选择器, 只显示当前账本的分类)
   - 确认信息: "将 N 条交易修改为 [分类名]"
4. 点击确定 → `PUT /transactions/batch-update` → `{ids: [...], category_id: "..."}`
5. 完成后提示成功/失败, 刷新表格
```
